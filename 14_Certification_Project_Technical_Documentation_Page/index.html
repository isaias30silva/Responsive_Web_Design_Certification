Introduction

HTML tags are the keywords on a web page that define how your web browser must format and display your web page.

Almost all tags contain two parts, an opening, and a closing tag. For example, <html> is the opening tag and </html> is the closing tag. Note that the closing tag has the same text as the opening tag, but has an additional forward-slash ( / ) character. 

There are a total of 100 HTML tags and here you will see and understand some of the most important HTML tags to use in your Webpages!

Comment tag

Definition

The comment tag is used to insert comments in the source code.

Comments are not displayed in the browsers. You can use comments to explain your code, which can help you when you edit the source code at a later date. This is especially useful if you have a lot of code. 

  
    <!-- Write your comments here -->

   <!-- Remember to add more information here -->

Why to use it

  • Explain code: Comments can help you understand your code when you edit it later.
  • Debug: You can comment out lines of code to search for errors
  • Hide content: You can use comments to hide parts of your code, such as a part of a paragraph.
  • Find where an element ends: You can put comments after closing tags to help you find where an element ends.
HTML tag

Definition

The <html> tag acts as the main container for all HTML elements, framing the entire web content structure. It follows the <!DOCTYPE> declaration and requires both opening and closing tags. 

  
  <!DOCTYPE html>
<html lang="en">
</html>

Why to use it

Everything still works because the browser is plugging it in for you. You should use it because it makes your code more clear and standard.

A developer looking at your code might be confused about what they're seeing at first because they would wonder where the <html> tag is.

As with any standard, the <html> tag guarantees that things will work. Currently the tag can be omitted, but I still wouldn't recommend it, to be safe.

Head tag

Definition

The head tag is an HTML element that defines the head section of an HTML document. It contains metadata, or data about data, that is used by browsers and search engines, but is not displayed on the page. 

  
  <!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8" />
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
<title>Here the Page Title/title>
<link rel="stylesheet" href="styles.css" />
</head>
<body>
</body>
</html>

Why to use it

The <head> element is a container for metadata (data about data) and is placed between the <html> tag and the <body> tag. Metadata is data about the HTML document. Metadata is not displayed. Metadata typically define the document title, character set, styles, scripts, and other meta information.

Body tag

Definition

The body tag in HTML is a container element that holds the main content of an HTML document and is visible on the web page: 

  
  <!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8" />
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
<title>Here the Page Title/title>
<link rel="stylesheet" href="styles.css" />
</head>
<body>
<h1>Welcome to My Webpage/h1>
<p>This is the content of my webpage/p>
</body>
</html>

Why to use it

The Body tag is an integral part of any HTML page, required to indicate the boundaries of the main content that is displayed to users.

Main tag

Definition

The HTML <main> tag is a structural element that indicates the primary content of a document or application 

  
  <!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8" />
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
<title>Here the Page Title/title>
<link rel="stylesheet" href="styles.css" />
</head>
<body>
<main>
<h1>Most Popular Browsers/h1>
<p>Chrome, Firefox, and Edge are the most used browsers today/p>
</main>
</body>
</html>

Why to use it

The <main> tag should be used to contain the content that is directly related to or expands upon the document's or application's central topic or feature. It should not contain any content that is repeated across documents such as sidebars, navigation links, copyright information, site logos, and search forms.
